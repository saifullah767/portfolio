# Database Details
## Overview
- Role: Database Administrator
- Focus: performance, reliability, backups, and data integrity
- Environment: production + staging clusters with HA/replication
## Core Practices
- Indexing strategy based on workload patterns and query plans
- Backup & recovery drills with defined RPO/RTO targets
- Security & access reviews with least-privilege policies
- Monitoring and alerting for latency, locks, and replication lag
## Reliability Targets
- Uptime: 99.95%
- Query latency: sub-200ms for critical endpoints
- Recovery: RPO < 15 minutes, RTO < 1 hour
## Tooling & Platforms
- PostgreSQL, MySQL, SQL Server
- Performance profiling, query analysis, and log review
- Schema governance and change management
## Database Selection Guide
- PostgreSQL: best for complex queries, analytics, and extensibility
- MySQL: great for high-read workloads and straightforward OLTP apps
- SQL Server: strong for enterprise reporting and Microsoft stack integration
- Oracle Database: ideal for large enterprise systems and regulated workloads
## SQL & Oracle Notes
- SQL: query clarity, predictable plans, maintainable schemas
- Oracle: optimizer, partitioning, RAC, enterprise features
## DBA Core Responsibilities
- Capacity planning and storage forecasting
- Availability design (replication, failover testing)
- Backup strategy (full/incremental, PITR, verification)
- Security posture (roles, auditing, encryption)
- Performance tuning (plans, indexes, caching)
- Maintenance windows (patching, upgrades)
## Operational Checkpoints
- Daily: alerts, replication lag, slow queries, logs
- Weekly: index bloat checks, restore tests, storage review
- Monthly: patch planning, schema review, capacity update
